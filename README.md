Out of nowhere I got a strong desire of building something powerful, meaningful and educational for myself. I had no choice but to create my own programming language.

I went straight up to Youtube and found [this wonderful playlist](https://youtube.com/playlist?list=PLUDlas_Zy_qC7c5tCgTMYq2idyyT241qs&si=Xyluj5YDMPhmQ14z), I headed to GitHub and [found gold too](https://github.com/DoctorWkt/acwj?tab=readme-ov-file).  Tho, I was not able to understand DoctorWkt's notes (they were too difficult for me to understand), I consider it a valuable resource. And what about the playlist? It was straight out perfect, but after a few videos I wanted to approach in a more traditional way, less flashy and a bit slower.

So, I got a book called Crafting Interpreters, by Robert Nystrom, and this became my main resource to learn what I need to.
This repository is the documentation of my journey, which goal is to bootstrap mei compiler in mei.

This is the roadmap of the steps I have taken so far, feel free to check them out.

2. [Episode 1](docs/ep1_wth.md): First hit on a wall: Wth I'm REALLY doing?
3. Episode 2: Learning how to create compilers by re-implementing jlox/clox
    1. [Episode 2.1](docs/ep2.1_scanners.md): First things first: Scanners and lexers
